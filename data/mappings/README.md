# Mappings

민원 목적 목록과 공공데이터 `taskNm` 매핑 테이블을 이 디렉터리에 둡니다.

## taskNm 정리 기준

- 기준 문서: `implements_roadmap.md` 4단계
- 기준 데이터: `data/recommend/task-name-samples.json`
- 현재 매핑 버전: `2026-04-stage-4`

이번 단계에서는 공공데이터포털 실시간 민원실 데이터에서 실제로 관측된
`taskNm` 값을 바탕으로, MVP에서 지원할 민원 목적 5종에 대해 포함 규칙과
제외 규칙을 정리했다. 목적 간 경계가 애매한 혼합 창구는 보수적으로 제외하거나
우선 목적을 하나로 고정했다.

## 지원 민원 목적

- `passport-reissue`: 여권 재발급
- `passport-pickup`: 여권 수령
- `certificate-issuance`: 제증명 발급
- `family-relation-certificate`: 가족관계 증명
- `resident-registration`: 주민등록 민원

## 목적별 분류 요약

### 1. 여권 재발급

- 대표 포함 `taskNm`
  - `4.여권신청`
  - `여권 신청`
  - `여권접수`
  - `신청서 작성후, 여권 신청`
  - `여권발급 신청하기`
- 대표 제외 `taskNm`
  - `여권교부`
  - `여권수령`
  - `여권 찾기`
  - `여권 심사•교부`
  - `여권신고`
  - `여권접수 교부인원`
  - `여권접수 대기인원`
- 분류 원칙
  - `신청`, `접수`, `발급 신청` 계열만 포함한다.
  - `교부`, `수령`, `찾기`, `심사`가 섞이면 제외한다.

### 2. 여권 수령

- 대표 포함 `taskNm`
  - `4.여권교부`
  - `여권교부`
  - `여권 수령`
  - `여권 찾기`
  - `여권 찾는곳`
  - `신청한 여권 찾기`
- 대표 제외 `taskNm`
  - `여권접수`
  - `여권발급 신청하기`
  - `여권 심사•교부`
- 분류 원칙
  - `교부`, `수령`, `찾기` 계열만 포함한다.
  - `접수`, `발급`, `심사`가 섞이면 제외한다.

### 3. 제증명 발급

- 대표 포함 `taskNm`
  - `6.통합제증명 발급`
  - `민원발급`
  - `제증명 발급`
  - `증명서 발급`
  - `통합제증명`
  - `통합증명발급`
  - `통합민원`
  - `제증명.민원접수`
- 대표 제외 `taskNm`
  - `지방세제증명`
  - `여권 수령, 증명서`
  - `주민등록•인감,가압관계 어디서나 민원`
- 분류 원칙
  - 일반 증명 발급 창구만 포함한다.
  - `가족관계`, `주민등록`, `여권`, `지방세`, `자동차`가 드러나면 제외한다.

### 4. 가족관계 증명

- 대표 포함 `taskNm`
  - `가족관계 발급•접수`
  - `가족관계등록신고`
  - `가족관계신고`
  - `출생,혼인/이혼,사망`
  - `혼인•이혼•개명•출생•사망`
  - `가압관계 신고(혼인,출생,이혼,사망등)`
- 대표 제외 `taskNm`
  - `등본,인감/가족관계`
  - `주민등록•인감,가압관계 어디서나 민원`
- 분류 원칙
  - `가족관계`, `혼인`, `이혼`, `출생`, `사망` 계열을 포함한다.
  - 실데이터 오탈자 `가압관계`는 `가족관계` 별칭으로 처리한다.
  - `등본`, `인감`, `주민등록`, `어디서나 민원`이 섞인 혼합 창구는 제외한다.

### 5. 주민등록 민원

- 대표 포함 `taskNm`
  - `주민등록•인감,가압관계 어디서나 민원`
  - `등본,인감/가족관계`
  - `통합제증명`
  - `통합증명발급`
- 대표 제외 `taskNm`
  - `민원발급`
  - `통합민원`
  - `지방세제증명`
- 분류 원칙
  - `주민등록`, `등본`, `초본`, `인감`, `전입` 계열을 우선 포함한다.
  - 혼합 창구 중 `주민등록` 또는 `등본/인감`이 드러나는 경우 주민등록 목적에서
    우선 허용한다.
  - `민원발급`, `통합민원`, `증명서 발급`처럼 범위가 넓은 항목은 애매 항목으로 둔다.

## 애매 항목 처리 원칙

- 아래와 같은 `taskNm`은 다른 목적과 겹치거나 범위가 넓어 보수적으로 처리한다.
  - `여권 심사•교부`
  - `여권신고`
  - `민원발급`
  - `통합민원`
  - `증명서 발급`
  - `등본,인감/가족관계`
  - `주민등록•인감,가압관계 어디서나 민원`
- 이 항목들은 목적별 `excludeRules` 또는 `ambiguousTaskNames`에 반영한다.
- 다음 단계에서 실데이터 표본이 더 늘어나면 우선순위를 다시 조정한다.
